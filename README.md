# AI 自駕車系統 - 第二階段成果報告

本專案為基於 Raspberry Pi 與 OpenCV 開發的 AI 智慧自駕小車系統。目前已完成第二階段開發，實現了精準的車道追隨、雙向紅綠燈識別、以及優化的 Web 遠端控制介面。

## 核心功能說明

### 1. 車道辨識與追隨 (detect.py)
- **視覺處理**：採用頂帽運算 (Top-hat) 與 Otsu 自動門檻法，有效提取白線特徵。
- **大彎道優化**：動態調整 ROI (感興趣區域) 至畫面下方的 60%-100% 區間，有效過濾遠景雜訊。
- **單線補償**：當偵測到單邊車道線時，系統自動計算虛擬中心，確保在彎道中不丟失路徑。

### 2. 雙向紅綠燈識別 (traffic.py)
- **鏡像辨識**：支援畫面左側與右側的紅綠燈偵測。
- **邏輯保護**：偵測到紅燈時，系統會記錄「等待狀態」並自動煞停；需偵測到綠燈訊號後方可起步前進。

### 3. 多功能 Web 控制面板 (index.html, app.py)
- **模式切換**：支援「手動駕駛」與「自動駕駛」即時切換。
- **遠端雲台**：整合雙軸雲台控制，可透過網頁調整鏡頭視角。
- **即時數據**：網頁面板會即時顯示車道偏移量 (Offset) 與目前的號誌辨識狀態。

### 4. 硬體驅動與測試 (motor_control.py, test.py)
- **麥克納姆輪驅動**：支援前進、後退、平移及原地旋轉。
- **測試工具**：提供馬達逐一測試腳本，確保硬體接線正確無誤。

## 檔案結構說明
- `app.py`: Flask Web 伺服器核心，處理影像串流與控制邏輯中樞。
- `detect.py`: 負責車道線 OpenCV 影像算法。
- `traffic.py`: 負責紅綠燈辨識與紅燈鎖定邏輯。
- `motor_control.py`: 底層馬達驅動與 PCA9685 PWM 訊號控制。
- `index.html`: Web 前端介面。
- `test.py`: 硬體初始化測試工具。
- `roboflow model`: 訓練之紅綠燈辨識模型(受限於容量未實際使用)。

## 操作指引

### 環境需求
- Python 3.9+
- OpenCV (opencv-python)
- Flask
- picamera2 (Raspberry Pi 專用)
- smbus (I2C 通訊)

### 啟動步驟
1. **硬體檢查**：執行 `python test.py` 確保四個馬達旋轉方向正確。
2. **啟動系統**：執行 `python app.py`。
3. **進入控制台**：在瀏覽器輸入 `http://[您的樹莓派IP]:5000`。
4. **自動駕駛**：在控制面板點擊「自動駕駛」按鈕，小車即開始執行車道追隨任務。
