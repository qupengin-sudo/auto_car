# Auto_Car_Project - Phase 1
**基於 Raspberry Pi 5 與 OpenCV 的全自動循線系統**

本專案為 E2 自動駕駛開發實作的第一階段。目標是讓配備鏡頭的自走車能透過 **OpenCV** 即時辨識賽道標線，並利用 **PID 控制演算法**，在具有高光反射的地板環境下實現穩定且平滑的全自動循線行駛。

## 核心技術說明

**1. 空間對比判定法 (Spatial Line Classification)**
系統採用動態的 **底部錨點 (Bottommost)** 辨識邏輯。透過計算線段最靠近畫面下方之點的 **X 座標位置**，並以影像中央為基準點，精確區分左右車道線。即使在標線斷裂或暫時丟失的情況下，也能維持辨識的強健性。

**2. 抗反光視覺預處理 (Anti-Reflection Processing)**
針對賽道打蠟地板與強烈白光產生的反光干擾，本專案整合了以下技術：
- **HSV 色彩過濾**：在 HSV 空間中進行標線顏色提取，比傳統 BGR 更能抵抗環境光變化。
- **頂帽運算 (Top-Hat Transform)**：利用形態學運算抑制大面積的亮斑雜訊，從高亮度背景中精確提取白色膠帶標線特徵。

**3. 物理盲區與延遲補償 (Delayed Compensation)**
由於鏡頭安裝夾角導致車頭正下方存在約 **17 公分** 的視覺盲區。我們實作了延遲補償邏輯，將單線修正的補償值從 45 像素精確調降至 **38 像素**，補足空間資訊落差，成功解決急彎時過早切彎的問題。

**4. PID 控制演算法 (PID Control)**
利用偏移量 (Offset) 計算左右輪速差，維持行駛穩定。
- **Kp (1.2)**：提供基礎修正力。
- **Kd (0.45)**：預測偏移變化量，有效抑制車頭在直線與進入彎道時的慣性震盪。

## 系統資料流 (Data Flow Diagram, DFD)

本系統資料流路遵循 **感知、運算、執行** 的循環：
1. **感知層**：透過 Picamera2 擷取原始影像。
2. **處理層**：**detect.py** 執行預處理、ROI 遮罩（限制於高度 0.6 ~ 0.95）及偏移量計算。
3. **運算層**：**app.py** 接收偏移量並透過 PID 公式計算馬達脈衝。
4. **執行層**：透過 I2C 通訊指揮 **PCA9685** 驅動馬達達成轉向。

## 檔案清單與功能說明

- **app.py**：系統核心，啟動 Flask 網頁伺服器、管理自動駕駛主迴圈及接收 Web 指令。
- **detect.py**：視覺模組，包含 HSV 過濾、頂帽運算、ROI 設定及左右標線分類邏輯。
- **motor_control.py**：硬體驅動層，封裝馬達前進、停止及雲台角度控制指令。
- **index.html**：遠端監控介面，提供即時影像串流與控制按鈕。

## 執行與操作

1. 確保 Raspberry Pi 5 已連接 Camera 與馬達驅動板。
2. 安裝必要套件：`pip install flask opencv-python numpy smbus2`。
3. 執行主程式：`python3 app.py`。
4. 於瀏覽器開啟 `http://<Pi_IP>:5000` 即可進入監控介面並啟動 **自動駕駛**。
